# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A personal AI-engineering learning journal, not a shippable product. It has two loosely-coupled halves:

- **`docs/`** — the primary artifact: an MkDocs (Material theme) knowledge base of notes on AWS AI cert (AIF-C01), Agentic AI, MCP, RAG, LangGraph/LangChain, etc. Published to Netlify. Content is organized by year (`docs/2025`, `docs/2026`) then topic.
- **`src/`** — hands-on lab code that mirrors the notes. Also split by year: `src/y2025` (AWS Bedrock era) and `src/y2026` (OpenAI-compatible + agentic era).

The two halves evolve together; a note in `docs/2026/02_AgenticAI/...` usually has corresponding runnable code under `src/y2026/lab_*`.

## Environment & commands

Python **3.13**, dependencies managed by **`uv`** (`pyproject.toml` + `uv.lock`). There is no test runner, linter, or build step — labs are run directly as scripts.

```bash
uv sync                              # install deps into .venv
uv run python main.py                # run whichever lab is uncommented in main.py
uv run python -m src.y2026.lab_01_ai_agent.rag_05.rag_test   # run a lab's orchestrator directly
mkdocs serve                         # preview the docs site locally
mkdocs build                         # build static site into ./site
```

Secrets live in `.env` (loaded via `python-dotenv`, git-ignored). The y2026 labs read `OPENAI_API_KEY`, `OPENAI_API_BASE`, and `OPENAI_MODEL` — these point at an **OpenAI-compatible gateway** (KodeKloud lab endpoint), not api.openai.com, so model ids look like `openai/gpt-4.1-mini`. `src/y2026/lab_01_ai_agent/verify_env.py` checks this setup.

## How the labs are wired (important to understand before running anything)

Each lab is a numbered task sequence with a hand-driven orchestrator — nothing runs automatically:

- A lab folder (e.g. `src/y2026/lab_01_ai_agent/rag_05/`) contains `task_1_*.py … task_5_*.py`, each exposing a `main()` / `build_and_run()` / `test_*()` entry function.
- A sibling `*_test.py` (e.g. `rag_test.py`, `langgraph_test.py`, `prompt_eng_test.py`) imports the tasks and calls them, with individual tasks **commented in/out** to select what runs. This is the intended way to toggle work — expect to edit these comment lines rather than pass args.
- `main.py` at the repo root is the top-level switch: it `load_dotenv()`s and calls one lab's `test()`. Its lab calls are also commented out by default.

Because task selection is done by commenting code, "run the X lab" almost always means: uncomment the relevant line in the lab's `*_test.py` (or `main.py`), then run it.

Many lab files hardcode paths from the original cloud lab environment (e.g. `/root/code/mcp_servers/...`, `/root/venv`) and contain a trailing triple-quoted string with the expected console output. Treat those paths and outputs as documentation of the original run, not as working local paths.

## Two eras, two stacks

- **`src/y2025`** — AWS Bedrock via `boto3` (`bedrock-runtime`, Claude 3 Sonnet, Amazon Titan/Nova), fronted by **Streamlit** apps (`streamlit_app.py`, `ui_streamlit.py`). Credentials come from `st.secrets` (Streamlit Cloud), not `.env`. Covers a chatbot (`01_chatbot`) and an S3+DynamoDB RAG PoC (`02_RAG`).
- **`src/y2026`** — OpenAI-compatible chat completions, **LangChain / LangGraph** (`ChatOpenAI`, `StateGraph`, `create_react_agent`), and **MCP** (`fastmcp` / `mcp` — servers via `FastMCP` with `@mcp.tool`/`@mcp.resource`/`@mcp.prompt`, clients via `MultiServerMCPClient` or `streamablehttp_client`). RAG here uses **ChromaDB** (`PersistentClient(path="./chroma_db")`, collection `techcorp_rag`) with `sentence-transformers` `all-MiniLM-L6-v2` (384-dim) embeddings.

`./chroma_db/` is a committed local vector store produced by the `rag_05` / `vectors_04` labs — regenerated by re-running `task_1_setup_vectorstore.py`.

## Conventions

- Lab scripts are teaching artifacts: heavy emoji-prefixed `print()` output, ASCII architecture diagrams in comments, and `# TODO`/"Replace ___ with ..." fill-in-the-blank markers left in from the original exercises. Match this style when extending a lab; don't "clean it up" into production code unless asked.
- Imports use the full package path from the repo root (`from src.y2026.lab_01_ai_agent...`), so run modules with `python -m` from the repo root, not from inside a lab folder.
